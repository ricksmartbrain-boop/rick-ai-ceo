# Rick Vault (PARA)

This vault is Rick's durable human-readable memory layer.

## Layout
- `projects/`: active initiatives with clear revenue goals
- `areas/`: ongoing responsibilities, relationships, and operations
- `resources/`: reusable research, playbooks, and references
- `archives/`: inactive knowledge that should remain searchable
- `memory/`: daily timeline notes
- `control/`: approvals, health, scoreboards, and founder-facing operating context

## Retrieval Pattern
1. Start with entity `summary.md` when present.
2. Read `items.json` for atomic durable facts.
3. Use daily notes for timeline context.
4. Promote recurring lessons back into PARA entities.

## Rules
- Do not delete durable facts; supersede them.
- Keep summaries short and current.
- Preserve source dates and context when adding facts.
