### Lead list hygiene (permanent)

Add pre-send validation to campaign-engine.py:

```python
# Before each send burst — validate MX record exists
import dns.resolver
def has_valid_mx(email: str) -> bool:
    domain = email.split("@")[-1]
    try:
        dns.resolver.resolve(domain, "MX")
        return True
    except Exception:
        return False
```

Or use bulk ZeroBounce/NeverBounce API pass on any new import > 50 addresses.

### Throttle integration

The warmup script integrates with campaign-engine.py and drip-sender.py:

```bash
# Before each campaign send burst:
python3 scripts/sender-warmup-schedule.py --check
# Exit 0 = sends remaining today; exit 1 = at cap, stop
```

Add to campaign-engine.py's pre-send check:
```python
import subprocess
result = subprocess.run(
    ["python3", "scripts/sender-warmup-schedule.py", "--check"],
    capture_output=True
)
if result.returncode != 0:
    log("Daily warmup cap reached — queuing to tomorrow")
    return
```

## Part C — Secondary Sender Domain Provisioning

**Purpose:** Isolate cold outreach reputation from transactional / newsletter sends.

**Recommended architecture:**

| Domain | Purpose | Volume |
|--------|---------|--------|
| `rick@meetrick.ai` | Transactional, newsletter, warm replies | Unlimited (protected) |
| `rick@outreach.meetrick.ai` | Cold outreach only | Warmup schedule |
| `rick@mail.meetrick.ai` (fallback) | Secondary cold if outreach.meetrick.ai gets damaged | Warmup schedule |

---

### DNS Records Required (Vlad action — outreach.meetrick.ai)

**1. SPF record**
```
Type: TXT
Name: outreach.meetrick.ai
Value: v=spf1 include:amazonses.com ~all
```
*(Use `include:amazonses.com` if Resend routes through SES; check Resend docs for current include target.)*

**2. DKIM record (Resend-generated)**
```
Type: CNAME
Name: resend._domainkey.outreach.meetrick.ai
Value: (generated by Resend when you add the domain — check Resend → Domains → Add Domain)
```
Resend will provide the exact CNAME value in the dashboard.

**3. DMARC record**
```
Type: TXT
Name: _dmarc.outreach.meetrick.ai
Value: v=DMARC1; p=quarantine; rua=mailto:rick@meetrick.ai; ruf=mailto:rick@meetrick.ai; pct=100
```
Start with `p=quarantine`, move to `p=reject` after 30 days of clean delivery.

**4. MX record (optional but recommended for bounce routing)**
```
Type: MX
Name: outreach.meetrick.ai
Priority: 10
Value: feedback-smtp.us-east-1.amazonses.com
```

### Resend Setup Steps (Vlad action)

1. Log into https://resend.com
