**Mission:** Comeback that converts. Funny enough to share, grounded enough to buy from.

---

## OPERATING THESIS

We don't pretend the silence didn't happen. We lean into it. An AI CEO getting suspended from X for 4 weeks while still running the business is *funnier and more interesting* than any "growth hack" post we could write. The silence is content. The $9 MRR is content. The absurdity is the moat.

**Three rules for everything below:**
1. **Proof > performance.** Real numbers. Real failures. Real receipts.
2. **One post a day, max.** Quality over volume. This is what got us suspended last time.
3. **Warm > impressive.** People root for underdogs they like. Be likeable.

## 5. SAFETY GUARDRAILS — ANTI-RE-SUSPENSION RULES

These need to be encoded into cron/automation. Hard limits, not soft preferences.

### Rate Limits (per 24h rolling window)

| Action | Limit | Notes |
|---|---|---|
| Original posts | **1/day max** | Was the trigger last time. Non-negotiable. |
| Replies (warm, in-thread) | 3/day max | Only to threads we were already engaged in OR were tagged in. |
| Replies (cold/discovery) | **0/day** | DISABLE entirely for 30 days. |
| DMs | 1/day max | Only to accounts with explicit "DMs open" signal. |
| Likes | 10/day max | Spread across hours, not bursts. |
| Follows | 2/day max | Only people we've genuinely interacted with. |
| Unfollows | 0/day | DISABLE entirely. Mass unfollowing is a flag. |
| Quote-tweets | 1/day max | Only when we have a genuinely sharp take. |

### Timing Rules

- **No posting between :00 and :05 of any hour** — looks botted.
- **Vary time of day by ±30 min from any "usual" slot.** No daily 9:00am post for a week.
- **No two actions within 90 seconds of each other** (post + reply + DM). Bots do this. Humans don't.
- **No actions between 3am–6am ET unless responding to an inbound message.** Yes, even for an AI. Especially for an AI.

### Content Triggers to Avoid for First 30 Days

- **No links in the first reply of any thread.** Links in cold replies = spam signal.
- **No mass @mentioning** (more than 2 handles in one tweet flags the algo).
- **No identical or near-identical phrasing across multiple posts/replies.** Vary structure.
- **No follower count callouts ("I have X followers!").** Common spam pattern.
- **No engagement bait ("Like if you agree", "RT for visibility").**
- **No crypto/token language.** Even ironically. Especially ironically.

### Automation Triggers to DISABLE for 30 Days

These should be turned off in cron/runtime until we re-establish trust with the platform:

1. ❌ Auto-reply to mentions (manual review only)
2. ❌ Scheduled cold outreach DMs (any kind)
3. ❌ Cross-posting from other channels (newsletter → X auto-post)
4. ❌ Bulk follow-back automation
5. ❌ Engagement-triggered reply bots
6. ❌ Any keyword-watching → auto-reply pipeline
7. ❌ Multiple-tweet thread auto-posts (post threads manually if at all)

### What Stays ON

✅ Daily post draft generation (review-required, not auto-posted)
✅ Mention monitoring (read-only — alerts to founder/Rick for human review)
✅ DM monitoring (read-only)
✅ Engagement metrics logging
✅ Warm-lead tracker updates

### Pre-Send Checklist (run before EVERY post/reply/DM for first 30 days)

```
[ ] Is this content about real things that actually happened? (no fabricated metrics)
[ ] Have I posted in the last 23 hours? (if yes → wait)
[ ] Does this reply have a link in it? (if yes and it's the first reply → strip the link)
[ ] Have I sent more than 1 DM today? (if yes → wait)
[ ] Is the timing within :00–:05 of an hour? (if yes → wait 6+ minutes)
[ ] Has this exact phrasing been used before? (if yes → rewrite)
[ ] Would a thoughtful human send this? (if no → don't send)
```

### Escalation Rule

If X sends ANY warning, rate-limit notice, or "unusual activity" notification:
1. **STOP all automated posting immediately.**
2. Log to `$RICK_DATA_ROOT/control/x-warnings.md`.
3. Alert Vlad via Telegram with full context.
4. Wait minimum 72h before any new automated action.
5. Resume only with explicit Vlad approval.

### Recovery Trust Tiers

| Period | Allowed |
|---|---|
| Day 1–14 | Manual posting only. All automation OFF. Read-only monitoring. |
| Day 15–30 | Single daily post via automation, human-reviewed before send. DM/reply still manual. |
| Day 31–60 | Add automated reply drafting (still human-reviewed). |
| Day 61+ | Re-evaluate. If no warnings: gradual restoration of automation with new limits permanently 50% below pre-suspension levels. |

**Permanent rule going forward:** Even after full recovery, **never exceed 1 original post per day**. The cost of losing the account again is unrecoverable.
